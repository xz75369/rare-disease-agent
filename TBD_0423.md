### 具体步骤

**Step 1.1：充值（如果还没充）**

- 访问 https://platform.deepseek.com
- 充值 50 元（够 48 小时用到爆）
- 生成 API key，记到密码管理器

**Step 1.2：本机测试**

```bash
pip install openai
```

新建 `test_api.py`：

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key="sk-你的key",
    base_url="https://api.deepseek.com"
)

# 测试 1：基础调用
resp = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": "用一句话介绍 Dravet 综合征"}]
)
print("✅ 基础调用成功")
print(resp.choices[0].message.content)

# 测试 2：JSON mode（项目里会大量用）
resp = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "Return JSON only."},
        {"role": "user", "content": '返回一个 JSON: {"disease": "Dravet", "gene": "SCN1A"}'}
    ],
    response_format={"type": "json_object"}
)
print("✅ JSON mode 成功")
print(resp.choices[0].message.content)

# 测试 3：中文长 prompt（模拟真实场景）
long_prompt = "以下是一个病例：2岁男童..." * 20
resp = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": long_prompt}],
    max_tokens=500
)
print("✅ 中文长 prompt 成功")
print(f"Token 消耗：{resp.usage.total_tokens}")
```

运行：`python test_api.py`

## 🟡 TASK 5：Prompt 亲自审阅（30 min）

### 为什么这是关键任务？

代码明天写得快，**Prompt 明天改起来慢**。Prompt 是你对项目影响最大的东西。

### 审阅重点

打开 `prompts.py`，**逐条确认**：

#### ✅ 结构类

- [ ] 每个 Prompt 分 System + User 清晰
- [ ] JSON schema 明确写出
- [ ] 有"Return ONLY valid JSON, no markdown fences"的强约束

#### ✅ 引用类

- [ ] 明确"只能引用 [N] 中 N 在 evidence 列表内的"
- [ ] 明确"Never invent URLs or citations"
- [ ] 报告 Prompt 要求每个事实性论断带 [N]

#### ✅ ACMG 术语类

- [ ] VUS 不能被描述为"致病"
- [ ] Pathogenic/Likely Pathogenic 有明确用法
- [ ] 提到"ACMG 分级严格按 Exomiser 输出，不要修改"

#### ✅ 方向特异性（根据你选的方向）

**如果选方向 A（鉴别诊断）**：
- [ ] Prompt 明确要求"对比分析"而非"独立诊断"
- [ ] 明确要求找出"关键鉴别点"和"pathognomonic 特征"
- [ ] 要求输出"患者匹配度"和"缺失信息"

**如果选方向 C（行动规划）**：
- [ ] Prompt 明确要求具体行动（不是"建议进一步检查"这种空话）
- [ ] 每条建议都有优先级和依据

### 发现问题怎么办？

**小问题**（措辞不准）→ 直接改
**大问题**（整体方向偏）→ 回去问 AI 顾问

## 🟡 TASK 6：准备 Demo 案例（30 min）

### 为什么要准备备用案例？

**防现场数据出问题**：
- 生信队友延迟交付 → 你没东西可 debug
- 主办方数据格式跟预期不一样 → 解析失败
- 现场网络挂 → 直接切 mock 案例

**准备 2-3 个典型案例**，存在 `data/samples/` 下。

### 推荐案例（方向 A 鉴别诊断）

每个案例都要有**明确的鉴别对**：

**案例 1：Dravet 综合征**
- 鉴别：Dravet vs GEFS+ vs SCN8A 癫痫性脑病
- 典型 HPO：HP:0002373（热性惊厥）、HP:0002123（肌阵挛）
- 变异：SCN1A 截短变异（Pathogenic）

**案例 2：Gaucher 病 1 型**
- 鉴别：Gaucher vs Niemann-Pick B vs 其他溶酶体贮积病
- 典型 HPO：HP:0001744（脾大）、HP:0000938（骨损害）
- 变异：GBA p.N370S（Pathogenic）

**案例 3：Rett 综合征**
- 鉴别：Rett vs Angelman vs CDKL5 脑病
- 典型 HPO：HP:0002540（行走能力退化）、HP:0001263（发育迟缓）
- 变异：MECP2 截短变异（Pathogenic）

### 数据来源

- **HPO 列表**：从 OMIM 或 HPO 官网查
- **基因变异**：ClinVar 里找典型 Pathogenic 变异
- **手编 Exomiser 输出**：按你 schemas 的格式填

- [ ] `data/samples/case_1_dravet.json` 存在
- [ ] `data/samples/case_2_gaucher.json` 存在
- [ ] `data/samples/case_3_rett.json` 存在（可选）
- [ ] 每个都能被 `DiagnosisInput.model_validate()` 成功解析
